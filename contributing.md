# Contributing

Thanks for contributing! Contributions of all kinds are welcome:
code, docs, tests, bug reports, reviews, examples, and design discussions.

## Ways to contribute
- Open an issue (bug, idea, question, proposal)
- Improve documentation or examples
- Add tests
- Fix bugs / add features via pull requests

## Workflow
1. Create/choose an issue (or open one to discuss the change)
2. Fork the repo and create a branch:
   - `feat/<short-name>` for features
   - `fix/<short-name>` for bug fixes
   - `docs/<short-name>` for documentation
3. Make a small, focused change
4. Run tests locally
